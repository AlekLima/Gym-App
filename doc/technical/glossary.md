# Gym App Glossary

## Terms

- **User**: A person who uses the application to track workouts, view routines, etc.
- **Admin**: An administrator with elevated privileges to manage users, routines, and system settings.
- **Workout Routine**: A structured set of exercises designed to achieve specific fitness goals.
- **Exercise**: A specific physical activity, such as a bench press or squat, with defined sets, reps, and weight.
- **Set**: A group of consecutive repetitions of an exercise.
- **Repetition (Rep)**: A single execution of an exercise.
- **Workout Session**: A single instance of performing a workout routine.
- **Progress Tracking**: The process of recording and viewing workout performance over time.

## Definitions

- **Authentication**: The process of verifying the identity of a user.
- **Authorization**: The process of determining what resources a user is allowed to access.
- **CRUD**: Create, Read, Update, Delete – the basic operations for managing data.
- **API**: Application Programming Interface, a set of rules that allows different software entities to communicate.
- **RESTful API**: An architectural style for designing networked applications, relying on stateless communication and standard HTTP methods.
